# Project Brief: Warp Zone Folio

## Название проекта
**Warp Zone Folio** - Pixel-Art Developer Portfolio

## Описание проекта
Современное портфолио разработчика с ретро-эстетикой, вдохновленное классическими 8-bit/16-bit игровыми интерфейсами. Одностраничное веб-приложение (SPA), оптимизированное для статического хостинга на GitHub Pages.

## Цели проекта

### Основные цели:
1. **Создание уникального визуального стиля** - дизайн с pixel-art эстетикой, выделяющийся среди типовых портфолио
2. **Полная функциональность без бэкенда** - статический сайт, не требующий серверной части
3. **Простота кастомизации** - вся персональная информация управляется через конфигурационные файлы
4. **Производительность и доступность** - быстрая загрузка, адаптивный дизайн, поддержка accessibility
5. **GitHub Pages ready** - готовность к деплою на GitHub Pages "из коробки"

### Дополнительные цели:
- Плавная навигация между секциями
- Поддержка светлой/темной темы с pixel-art стилизацией
- Интерактивные анимации (звездное поле, пиксельные эффекты)
- Семантическая разметка для SEO

## Целевая аудитория

### Основная аудитория:
- **Рекрутеры и HR** - ищут портфолио с проектами и навыками
- **Технические менеджеры** - оценивают технический стек и опыт
- **Потенциальные клиенты** - ищут веб-разработчика для проектов

### Вторичная аудитория:
- **Другие разработчики** - могут использовать как шаблон для своего портфолио
- **Сообщество open source** - проект является шаблоном для повторного использования

## Ключевые требования

### Функциональные требования:
1. **Секции портфолио:**
   - Hero (приветствие, имя, tagline)
   - About (описание разработчика)
   - Skills (категоризированные навыки с уровнями)
   - Projects (проекты с фильтрацией featured)
   - Timeline (опыт работы и образование)
   - Contact (контактная информация)
   - Footer

2. **Навигация:**
   - Фиксированная навигационная панель
   - Smooth scroll к секциям
   - Автоматическое определение активной секции
   - Мобильное меню

3. **Темизация:**
   - Переключение между светлой и темной темой
   - Сохранение выбора в localStorage
   - Pixel-art стилизация для обеих тем

4. **Адаптивность:**
   - Полностью адаптивный дизайн (mobile-first)
   - Оптимизация для всех размеров экранов
   - Pixel-perfect верстка

### Нефункциональные требования:
1. **Производительность:**
   - Быстрая загрузка (оптимизация bundle size)
   - Плавные анимации (60 FPS)
   - Ленивая загрузка изображений

2. **Доступность:**
   - Семантическая HTML разметка
   - ARIA labels для интерактивных элементов
   - Поддержка клавиатурной навигации
   - Контрастность цветов (WCAG AA)

3. **SEO:**
   - Метатеги (title, description, og:tags)
   - Структурированные данные
   - Оптимизация для поисковиков

4. **Поддерживаемость:**
   - Модульная структура кода
   - Централизованная конфигурация
   - Понятное именование компонентов
   - TypeScript для типобезопасности

## Ограничения

### Технические ограничения:
- **Только фронтенд** - нет серверной части, базы данных
- **Статический хостинг** - должен работать на GitHub Pages
- **Hash routing** - использование HashRouter из-за ограничений GitHub Pages
- **Размер bundle** - оптимизация для быстрой загрузки

### Дизайн ограничения:
- **Pixel-art стиль** - sharp corners (border-radius: 0)
- **Ограниченная цветовая палитра** - для аутентичного ретро ощущения
- **Chunky borders** - толстые границы (3px) для pixel-эффекта
- **Pixel shadows** - блочные тени вместо плавных

### Контентные ограничения:
- Вся информация управляется через конфигурационные файлы
- Плейсхолдеры для персональной информации ([YOUR_NAME], [EMAIL_ADDRESS], и т.д.)
